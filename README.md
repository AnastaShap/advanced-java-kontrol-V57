# advanced-java-kontrol-V57
Контрольна робота з дисципліни Advanced Java. Варіант: V57 (P3 + R10)

## Індивідуальні умови варіанта:

- Профіль оплати P3: CardPayment: дозволяє суму до 30_000; PayPalPayment: приймає суму від 300; BankTransferPayment: комісія 2%.
- Пакет правил R10: валідація - мінімум 2 позиції у замовленні.
- Знижки/комісії: знижка 5% при різних категоріях товарів >= 3.
- Переходи станів: NEW -> PAID -> SHIPPED -> DELIVERED, повторна оплата заборонена.
- Додатковий крок Template Method: реалізувати крок validateCategoryMix.
- Обов'язковий негативний сценарій: передбачити і протестувати виняток CategoryMixException

**Примітка щодо структури репозиторію:**
Через локальні налаштування середовища розробки головною гілкою було автоматично призначено master замість main.
Це спричинило певні технічні складнощі при створенні Pull Request.
Прошу звернути увагу, що фінальний код знаходиться в гілці *master*. Дякую за розуміння!
